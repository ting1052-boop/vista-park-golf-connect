# VISTA 입구 키오스크 화면 기획서

## 1. 목적

입구 안드로이드 태블릿을 무인 입장용 키오스크로 사용합니다.

이 화면은 관리자용이 아니라 고객용입니다. 고객이 매장에 도착했을 때 예약 확인 또는 현장 이용을 직접 진행하고, 입장 완료 후 해당 타석 장비 제어가 자동으로 실행되도록 안내합니다.

예상 경로:

```text
/kiosk/entrance
```

운영 방식:

- 안드로이드 태블릿 Chrome PWA 설치
- 또는 Fully Kiosk Browser로 고정 URL 실행
- 태블릿은 입구 또는 로비에 상시 전원 연결
- 관리자 기능, 설정 기능, 매출 화면은 노출하지 않음

## 2. 핵심 원칙

1. 한 화면에 너무 많은 선택지를 두지 않습니다.
2. 버튼은 크게 만들고, 문장은 짧게 작성합니다.
3. 장년층도 사용할 수 있도록 숫자 입력과 확인 버튼을 명확히 구분합니다.
4. 결제 연동 전에는 `결제 완료 처리` 버튼으로 현장 입장 흐름을 검증합니다.
5. 출입문 제어는 1차 범위에서 제외합니다.
6. 입장 완료 후 장비 제어는 Home Assistant 스크립트 호출 구조로 연결합니다.

## 3. 첫 화면

화면 제목:

```text
VISTA Park Golf Connect
비스타파크골프 시흥점
```

중앙 문구:

```text
오늘 이용을 시작할까요?
```

주요 버튼:

```text
[예약하고 왔어요]
[예약 없이 바로 이용할게요]
```

하단 정보:

```text
매장 전화 031-431-7050
문제가 있으면 매장에 연락해주세요.
```

디자인 기준:

- 배경: 밝은 회색
- 카드: 흰색
- 포인트: VISTA 그린
- 버튼 높이: 72px 이상
- 제목: 32px 이상
- 본문: 20px 이상
- 버튼 텍스트: 22px 이상

## 4. 예약 고객 입장 흐름

흐름:

```text
예약하고 왔어요
→ 전화번호 뒤 4자리 입력
→ 오늘 예약 조회
→ 예약 선택
→ 입장하기
→ 타석 배정 및 장비 준비
→ 완료 안내
```

전화번호 입력 화면:

```text
예약 확인

전화번호 뒤 4자리를 입력해주세요.

[ 1 ] [ 2 ] [ 3 ]
[ 4 ] [ 5 ] [ 6 ]
[ 7 ] [ 8 ] [ 9 ]
[ 지움 ] [ 0 ] [ 확인 ]
```

예약 조회 결과:

```text
예약을 확인했습니다.

14:00
2명
60분 + 10분
A-01 타석

[입장하기]
[처음으로]
```

입장 처리 중 화면:

```text
입장 처리 중입니다

조명 준비 중
PC 전원 켜는 중
프로젝터 준비 중
타석 배정 중

잠시만 기다려주세요.
```

입장 완료 화면:

```text
입장이 완료되었습니다

배정 타석: A-01
이용시간: 14:00 ~ 15:10

장비를 준비하고 있습니다.
타석으로 이동해주세요.
```

## 5. 예약 없는 현장 이용 흐름

흐름:

```text
예약 없이 바로 이용할게요
→ 인원 선택
→ 이용시간 선택
→ 빈 타석 확인
→ 요금 확인
→ 결제 완료 처리
→ 타석 배정 및 장비 준비
→ 완료 안내
```

인원 선택:

```text
몇 명이 이용하시나요?

[1명] [2명]
[3명] [4명]
[5명] [6명]
```

이용시간 선택:

```text
이용시간을 선택해주세요.

[30분 / 6,000원]
[60분 + 10분 / 10,000원]
[90분 / 16,000원]
[120분 / 20,000원]
```

확인 화면:

```text
이용 내용을 확인해주세요

매장: 비스타파크골프 시흥점
인원: 2명
이용시간: 60분 + 10분
요금: 10,000원
배정 방식: 빈 타석 자동 배정

[결제 완료 처리]
[처음으로]
```

1차 MVP에서는 실제 결제 연동 대신 `결제 완료 처리` 버튼으로 흐름을 검증합니다. 실제 카드 결제, 간편결제, 영수증 출력은 2차 기능으로 분리합니다.

## 6. 타석 장비 제어 연결

입장 완료 시 내부적으로 다음 작업이 실행되어야 합니다.

```text
1. 빈 타석 조회
2. access_sessions 생성
3. kiosk_sessions 생성
4. bays.status = in_use
5. Home Assistant script 호출
6. automation_logs 기록
```

타석별 스크립트 예:

```text
A-01 → script.bay1_on
A-02 → script.bay2_on
A-03 → script.bay3_on
```

1번 타석 ON 구성 예:

```text
script.bay1_on
- 공용 조명 ON
- 필요 시 AC ON
- 1번 PC WOL
- 1번 프로젝터 ON
- 1번 리시버 ON
```

2번 타석 ON 구성 예:

```text
script.bay2_on
- 공용 조명 ON
- 필요 시 AC ON
- 2번 PC WOL
- 2번 프로젝터 ON
```

3번 타석 ON 구성 예:

```text
script.bay3_on
- 공용 조명 ON
- 필요 시 AC ON
- 3번 PC WOL
- 3번 프로젝터 ON
```

## 7. 공용 장비 제어 기준

공용 장비는 타석별로 무조건 OFF하면 안 됩니다.

예:

```text
1번 타석 이용 중
2번 타석도 이용 중
1번 타석이 먼저 종료됨
→ 공용 조명과 AC는 유지
```

기준:

```text
첫 입장 → script.shared_on
중간 입장 → 해당 타석 script만 ON
마지막 퇴장 → script.shared_off
```

현재 VISTA 자동화 구조의 `access_sessions` 활성 개수 판단을 활용합니다.

## 8. 예외 화면

예약 없음:

```text

전화번호 뒤 4자리를 다시 확인해주세요.

[다시 입력]
[처음으로]
```

빈 타석 없음:

```text

잠시 후 다시 시도해주세요.
또는 매장에 문의해주세요.

[처음으로]
```

장비 준비 지연:

```text
장비 준비에 시간이 걸리고 있습니다

타석으로 이동 후 화면이 켜질 때까지 기다려주세요.

[확인]
```

네트워크 오류:

```text
연결 상태를 확인하고 있습니다

잠시 후 다시 시도해주세요.
문제가 계속되면 매장에 연락해주세요.

[다시 시도]
```

## 9. API 초안

예약 고객 입장:

```text
POST /api/kiosk/reservation/check-in
```

요청:

```json
{
  "storeId": "store-id",
  "phoneLast4": "1234",
  "reservationId": "reservation-id"
}
```

현장 이용 시작:

```text
POST /api/kiosk/walk-in/start
```

요청:

```json
{
  "storeId": "store-id",
  "partySize": 2,
  "durationMinutes": 60,
  "paymentStatus": "mock_paid"
}
```

응답:

```json
{
  "ok": true,
  "bayCode": "A-01",
  "startsAt": "2026-07-02T14:00:00+09:00",
  "endsAt": "2026-07-02T15:10:00+09:00",
  "automationStatus": "requested"
}
```

## 10. DB 반영 방향

현장 입장에서는 다음 테이블을 사용합니다.

```text
reservations
access_sessions
kiosk_sessions
bays
automation_logs
```

결제 연동 전에는 결제 테이블을 만들지 않고, access session 또는 reservation memo에 `mock payment`를 기록할 수 있습니다.

결제 연동 단계에서는 다음 테이블을 추가합니다.

```text
payments
- id
- store_id
- reservation_id
- access_session_id
- amount
- method
- status
- paid_at
- pg_provider
- pg_transaction_id
```

## 11. 구현 순서

1. Home Assistant에서 1번 PC WOL 성공
2. Home Assistant에서 `script.bay1_on` 성공
3. VISTA 무인제어 테스트 버튼에서 `script.bay1_on` 호출 성공
4. `/kiosk/entrance` 화면 구현
5. 예약 고객 전화번호 뒤 4자리 조회 구현
6. 현장 이용 mock 결제 흐름 구현
7. 입장 완료 시 access session 생성
8. 입장 완료 시 타석 상태 변경
9. 입장 완료 시 Home Assistant script 호출
10. 태블릿 PWA 또는 Fully Kiosk Browser에서 현장 테스트

## 12. 1차/2차 범위 구분

### 1차 키오스크 MVP

- 입구 태블릿용 화면
- 예약 고객 전화번호 뒤 4자리 조회
- 현장 이용 인원/시간 선택
- 결제 완료 mock 버튼
- 빈 타석 자동 배정
- 타석 상태 변경
- Home Assistant script 호출 구조
- 완료/오류 안내 화면

### 2차 기능

- 실제 카드 결제
- QR 예약 인증
- 카카오/네이버 로그인 기반 입장
- 영수증 출력
- 직원 호출
- 현장 결제 취소/환불
- 자동문/출입문 제어
- 웹 푸시 또는 문자 알림
