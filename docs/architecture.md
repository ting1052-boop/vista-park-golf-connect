# VISTA Park Golf Connect 설계 문서

## 1. 최종 제품 방향

VISTA Park Golf Connect는 HH Square가 운영·공급하는 스크린파크골프 가맹점을 위한 웹 기반 매장관리 프로그램입니다.

1차 MVP의 중심은 경기 내부 연동이 아니라 예약, 현장 입장, 타석 배정, 키오스크 이용시간, 장비 상태, 무인 운영 로그입니다. 매장주는 PC 관리자 화면에서 운영 상태를 관리하고, 고객은 모바일 웹/PWA에서 예약하거나 현장 키오스크를 통해 입장합니다.

- 고객: 모바일 웹/PWA에서 예약, 예약 확인, 현장 입장 요청, 이용시간 확인
- 매장주/직원: PC 관리자 화면에서 예약, 입장 세션, 타석, 장비, 키오스크 시간을 관리
- 본사관리자: 가맹점별 운영 상태, 장비 온라인 상태, 예약/입장/제어 로그를 확인

## 2. 1차 MVP 범위

### 고객 기능

- 카카오/네이버 계정 기반 회원가입 및 로그인 준비
- 모바일 예약 신청
- 예약 확인 및 취소 안내
- 예약 없는 현장 입장 요청
- 이용시간 및 예약 상태 확인

### 매장 관리자 기능

- 예약 현황 관리
- 현장 입장 승인
- 타석 배정
- 키오스크 세션 시작, 연장, 종료
- 조명, 냉난방기, 키오스크, 타석 전원 상태 관리
- 장비 오프라인/주의 상태 확인
- 노쇼 및 미입장 관리
- 운영 로그 및 HTML 리포트 확인

### 본사 관리자 기능

- 가맹점 관리
- 매장별 예약/입장 현황 확인
- 매장별 무인 운영 상태 확인
- 매장별 장비 등록 상태 확인
- 제어 로그와 운영 리포트 확인

## 3. 무인 운영 흐름

### 온라인 예약 고객

1. 고객이 모바일에서 매장, 날짜, 시간, 인원을 선택합니다.
2. 예약 10분 전 자동화 규칙이 실행됩니다.
3. 로비 조명, 냉난방기, 해당 타석 조명, 타석 키오스크 전원을 준비합니다.
4. 고객이 키오스크에서 예약 QR 또는 전화번호로 입장 인증을 합니다.
5. 인증이 완료되면 타석과 키오스크 세션이 시작됩니다.
6. 인원수 또는 업주가 지정한 시간 기준으로 이용시간 카운트다운이 시작됩니다.
7. 종료 시간이 지나면 키오스크가 잠기고 타석 조명, 냉난방기, 장비 전원이 정리됩니다.
8. 모든 제어 요청과 결과는 자동화 로그로 저장됩니다.

### 예약 없는 현장 고객

1. 고객이 로비 키오스크에서 빈 타석을 확인합니다.
2. 전화번호 또는 간편 로그인으로 현장 입장을 요청합니다.
3. 매장 정책에 따라 즉시 승인 또는 관리자 승인 대기 상태가 됩니다.
4. 승인되면 타석 조명, 냉난방기, 키오스크 세션이 시작됩니다.
5. 이용 종료 후 자동 정리 규칙이 실행됩니다.

## 4. 출입문 제어 범위

자동문 ON/OFF는 안전, 비상탈출, 현장 책임, 오작동 대응 기준이 필요하므로 1차 MVP에서는 제외합니다.

1차에서는 출입문을 직접 제어하지 않고, 고객의 입장 인증과 매장 내부 장비 제어까지만 다룹니다. 출입문 제어는 다음 조건이 정리된 뒤 2차 기능으로 검토합니다.

- 비상 개방 및 수동 개방 장치 확인
- 화재·정전·통신 장애 상황의 기본 동작 기준
- 관리자 원격 제어 권한과 책임 범위
- 제어 실패 시 알림 및 현장 대응 절차
- 모든 개방/차단/실패 이력의 감사 로그 저장

## 5. 헤이홈/Tapo 연동 방향

1차에서는 특정 제조사 API에 강하게 묶지 않고 자동화 어댑터 구조로 설계합니다.

지원 후보:

- `hejhome`: 헤이홈 스마트 스위치, 플러그, IR 리모컨
- `tapo`: Tapo 스마트 플러그, 조명, 허브
- `smartthings`: SmartThings 자동화 허브
- `ifttt`: Webhook 기반 자동화
- `home_assistant`: 로컬 자동화 서버
- `matter`: Matter 호환 장비
- `webhook`: 범용 HTTP 호출
- `manual`: 실제 장비 연결 전 수동 처리

1차 화면에서는 장비 등록, 상태 표시, 수동 ON/OFF, 자동화 로그를 먼저 구현합니다. 실제 헤이홈/Tapo 제어는 테스트 매장의 장비 ID와 Webhook URL이 준비되면 연결합니다.

## 7. 주요 DB 테이블

예약/회원/매장:

- `users`: Supabase Auth와 연결되는 사용자 및 역할
- `stores`: 가맹점 정보
- `store_users`: 매장별 관리자/직원 배정
- `members`: 고객 회원 정보, 소셜 로그인 식별자, 전화번호 해시
- `reservations`: 온라인 예약 및 현장 접수 예약
- `bays`: 타석 정보
- `business_hours`: 매장 영업시간
- `no_show_logs`: 노쇼 및 미입장 이력

무인 자동화:

- `automation_devices`: 조명, 냉난방기, 키오스크, 스마트 플러그, 라우터 등 자동화 장비
- `automation_scenes`: 예약 준비, 이용 시작, 이용 종료, 마감, 긴급 정지 같은 자동화 장면
- `automation_scene_steps`: 장면별 장비 제어 순서
- `access_sessions`: 고객 입장과 이용 세션
- `kiosk_sessions`: 키오스크 이용 가능 시간과 잠금 상태
- `automation_logs`: 모든 장비 제어 요청과 결과 로그

리포트/확장:

- `reports`: HTML 리포트 저장
- `store_notices`: 매장 공지
- `future_game_scores`: 향후 게임 프로그램 연동용 점수 저장 슬롯

## 9. 현재 구현 상태

현재는 제출용 1차 MVP mock UI와 Supabase 스키마 초안입니다.

- `/admin/dashboard`: 무인 매장 운영 대시보드
- `/admin/automation`: 조명, 냉난방, 키오스크, 타석 전원 제어 화면
- `/member/app`: 회원 예약/현장 입장용 모바일 화면
- `supabase/schema.sql`: 실제 DB 연결을 위한 테이블과 RLS 초안
- `supabase/seed.sql`: 시흥점 기준 샘플 데이터

실제 헤이홈/Tapo 장비 제어는 아직 연결 전입니다. 다음 단계에서 Webhook, Home Assistant, SmartThings, 또는 제조사 연동 방식을 선택해 `automation_devices`와 `automation_logs`에 연결합니다.
