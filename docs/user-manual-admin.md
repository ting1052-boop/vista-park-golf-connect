# VISTA Park Golf Connect 관리자 사용 설명서

버전: 1.0 (1차 MVP) · 작성일: 2026-07-02
대상: 본사관리자, 매장관리자, 직원

## 1. 시작하기

### 1.1 접속

PC 웹 브라우저(Chrome 권장)에서 아래 주소로 접속한다.

- 운영: `https://vista-park-golf-connect.vercel.app/admin/dashboard`

로그인하지 않은 상태로 관리자 화면에 접속하면 자동으로 로그인
화면으로 이동한다.

### 1.2 로그인

1. 이메일과 비밀번호를 입력하고 **로그인** 버튼을 누른다.
2. 로그인에 성공하면 원래 접속하려던 화면(기본: 대시보드)으로
   이동한다.
3. 비밀번호가 틀리면 "이메일 또는 비밀번호가 올바르지 않습니다"
   안내가 표시된다.

> 관리자 계정은 본사에서 발급한다. 계정 발급·비밀번호 초기화는
> 본사 운영팀(Supabase 콘솔 관리자)에 요청한다.

### 1.3 로그아웃

왼쪽 메뉴 하단의 **로그아웃** 버튼을 누르면 세션이 종료되고
로그인 화면으로 돌아간다. 공용 PC에서는 사용 후 반드시 로그아웃한다.

## 2. 대시보드

접속 주소: `/admin/dashboard`

- **현재 이용 중 / 입장 대기 / 사용 가능 / 점검 카드** — 타석
  상태별 집계. 타석 상태가 바뀌면 화면이 실시간 갱신된다.
- **확인 필요한 운영 알림** — 노쇼 확인, 입장 인증 대기, 장비
  전원 확인 등 조치가 필요한 항목. **확인 처리** 버튼으로 정리한다.
- **빠른 실행** — 입장 세션 시작, 매장 준비 ON, 타석 배정,
  제어 로그 출력.

## 3. 매장관리

접속 주소: `/admin/stores`

- **신규 등록**: 왼쪽 폼에 매장코드(예: VISTA-SH), 매장명, 주소,
  전화번호, 타석수, 운영상태를 입력하고 **Supabase에 추가**를 누른다.
- **수정/삭제**: 목록의 각 행에서 **수정** / **삭제** 버튼 사용.
- 타석수를 입력하면 해당 수만큼 타석관리에 타석 레코드가 자동
  생성·동기화된다.

## 4. 타석관리

접속 주소: `/admin/bays`

- 타석코드(예: A-01), 표시명, 상태, 메모를 관리한다.
- 상태 값: `available`(사용 가능) / `in_use`(이용 중) /
  `waiting`(입장 대기) / `maintenance`(점검).
- 타석 상태는 고객 예약 화면의 "예약 가능 타석" 계산에 바로
  반영된다. 점검이 필요한 타석은 `maintenance`로 바꾸면 고객이
  예약할 수 없다.

## 5. 장비관리

접속 주소: `/admin/devices`

- 매장 내 장비(프로젝터, 리시버, 키오스크 등)의 코드, 이름, 종류,
  상태, 최근 점검일을 관리한다.

## 6. 회원관리

접속 주소: `/admin/members`

- 닉네임, 전화 뒤 4자리, 가입경로, 연령대, 메모를 관리한다.
- 개인정보 최소 수집 원칙에 따라 전체 전화번호는 저장하지 않는다.

## 7. 예약관리

접속 주소: `/admin/reservations`

- 고객 앱 예약과 전화/현장 예약을 한 목록에서 관리한다.
- **등록**: 예약자명, 시작 시각, 타석코드, 인원, 채널, 상태를
  입력해 현장/전화 예약을 직접 추가할 수 있다.
- **수정**: 타석 재배정, 상태 변경(확정/입장/취소/노쇼)에 사용한다.
- 같은 타석의 시간이 겹치게 배정하면 저장되지 않고
  "선택한 타석과 시간대가 이미 예약되어 있습니다" 안내가 표시된다.
  이는 데이터베이스 차원의 안전장치로, 화면을 우회해도 겹침 예약은
  만들어지지 않는다.
- 90분 이상 또는 5명 이상 고객 예약은 "승인 대기" 상태로 접수된다.
  내용 확인 후 상태를 확정으로 변경한다.

## 8. 무인제어

접속 주소: `/admin/automation`

- 자동화 시나리오 4종: **예약 입장 준비** (로비 조명·냉난방·타석
  조명·키오스크 ON) / **타석 이용 시작** / **이용 종료 정리** /
  **마감 전체 OFF**.
- **시흥점 쇼룸 실기기 테스트**: 헤이홈·Tapo 장비를 대상으로 예약
  준비 → 이용 시작 → 종료 정리 흐름을 검증하는 패널. 실행에는
  테스트 비밀번호가 필요하다.
- 예약 5분 전 자동 준비는 서버 API(`/api/automation/reservation-prepare`)
  가 담당하며, 처리 결과는 예약별 `automation_prepare_status`로
  기록된다. 실제 장비 제어는 Home Assistant 환경변수
  (`HOME_ASSISTANT_URL`, `HOME_ASSISTANT_TOKEN`) 설정 후 활성화된다.
- 출입문 제어는 안전 검토 후 도입 예정(1차 제외).

## 9. 기타 화면

경기기록 / 랭킹 / 대회운영 / 조인모집 / 리포트 화면은 1차 MVP에서
화면 구조가 준비된 상태이며, 실 데이터 연동은 2차 확장에서
제공된다.

## 10. 자주 묻는 문제

| 증상 | 조치 |
| --- | --- |
| 관리자 화면이 로그인으로 계속 돌아감 | 계정 미발급 또는 세션 만료. 본사에 계정 확인 요청 후 재로그인 |
| 고객 화면에 "예약 가능 타석 0개" | 해당 시간대에 모든 타석이 예약됨(정상) 또는 타석 상태가 점검/이용중인지 타석관리에서 확인 |
| 예약 저장 시 "이미 예약되어 있습니다" | 다른 예약과 시간이 겹침. 타석 또는 시간을 바꿔 저장 |
| 매장 저장이 안 됨 | 매장코드 중복 여부 확인 (매장코드는 고유값) |
| 장비 자동 제어가 동작하지 않음 | Home Assistant 환경변수 설정 여부 확인 (2차 실증 항목) |
