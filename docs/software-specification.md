# VISTA Park Golf Connect 프로그램 명세서

버전: 1.0 (1차 MVP) · 작성일: 2026-07-02

## 1. 시스템 개요

스크린파크골프 가맹점을 위한 웹 기반 통합 운영 프로그램.
관리자는 PC 웹 브라우저로, 고객은 모바일 PWA로 사용한다.
모든 UI는 한국어이며, 장년층 고객도 사용할 수 있도록 큰 버튼과
단계형 화면으로 구성한다.

## 2. 사용자 구분

| 사용자 | 접속 경로 | 인증 |
| --- | --- | --- |
| 본사/매장 관리자 | `/admin/*` (PC 웹) | Supabase Auth 이메일/비밀번호. 미인증 시 `/admin/login`으로 자동 이동 |
| 고객 | `/member/app` (모바일 PWA) | 비회원 예약 (이름 + 전화번호 뒤 4자리만 수집) |

## 3. 고객 예약 화면 (`/member/app`)

### 3.1 화면 구성 (단일 페이지, 5단계 섹션)

1. **예약 매장** — 매장 드롭다운 선택. 선택 매장의 주소, 전화번호,
   운영시간, 예약 가능 타석 수 표시.
2. **예약자 정보** — 이름(또는 닉네임), 전화번호 뒤 4자리 입력.
   개인정보 최소 수집 원칙에 따라 전체 전화번호는 수집하지 않는다.
3. **날짜와 시간** — 오늘/내일/모레 3일 선택. 시간대 버튼
   (09:30~19:00, 9개 슬롯). 지난 시간은 취소선 비활성화, 해당 시간에
   배정 가능한 타석이 없으면 "마감" 표기 후 비활성화.
4. **인원과 이용시간** — 인원 1~6명. 이용시간과 요금:

   | 이용시간 | 요금 | 비고 |
   | --- | --- | --- |
   | 30분 | 6,000원 | |
   | 60분 | 10,000원 | 서비스 10분 추가, 실제 70분 구간 배정 |
   | 90분 | 16,000원 | 매장 승인 후 확정 |
   | 120분 | 20,000원 | 매장 승인 후 확정 |

   90분 이상 또는 5명 이상 예약은 자동확정이 아닌 승인 대기
   (`approval_required`) 상태로 접수된다.
5. **예약 확인** — 제출 전 매장/주소/날짜/시간/이용시간/인원/타석/
   예상요금/취소 안내를 요약 카드로 표시. 하단 고정 버튼이
   "예약 내용 확인" → "예약 확정" 2단계로 동작하여 오입력을 방지한다.

### 3.2 예약 처리 규칙

- 타석 배정: "자동 배정"(기본) 또는 직접 선택. 자동 배정은 해당
  시간대에 겹치는 예약이 없는 타석 중 첫 번째를 선택한다.
- 이중예약 방지 3중 구조:
  1. 화면: 시간대별 가용 타석 수를 실시간 계산하여 마감 시간대를
     선택 불가 처리
  2. 제출 직전: 서버에 겹침 재조회 (동시 제출 경합 방어)
  3. 데이터베이스: `reservations_bay_time_excl` 배타 제약조건이
     동일 타석의 시간 구간 겹침을 최종 차단. 위반 시 오류코드
     23P01을 한국어 안내문으로 변환하여 표시
- 지난 시각 예약 차단 (Asia/Seoul 타임존 기준 계산)
- 예약 확정 시 `status=confirmed`(자동확정) 또는 `requested`(승인제),
  `channel=member_app` 으로 저장
- 예약자 정보는 `guest_name`(이름), `guest_phone_last4`(뒤 4자리)
  컬럼에 분리 저장
- 예약 현황 목록에는 개인정보 보호를 위해 예약자 이름을 표시하지
  않는다 (시간·타석·인원·상태만 표시)

### 3.3 PWA

- `manifest.webmanifest` 제공: 앱 이름 "VISTA Park Golf Connect",
  단축 이름 "VISTA Connect", standalone 표시 모드, 시작 URL `/member/app`
- 아이콘: 192/512 PNG, maskable 512 PNG, apple-touch-icon 180 PNG
- 서비스 워커(`sw.js`): 네트워크 우선 + 캐시 폴백 전략
- Android Chrome "앱 설치", iPhone Safari "홈 화면에 추가" 지원

## 4. 관리자 화면 (`/admin/*`)

### 4.1 공통

- 좌측 고정 메뉴 12종: 대시보드, 예약관리, 무인제어, 매장관리,
  타석관리, 장비관리, 회원관리, 경기기록, 랭킹, 대회운영, 조인모집,
  리포트
- 밝은 회색 배경 + 흰색 카드형 업무 UI, VISTA 그린 포인트
- 미인증 접근 시 로그인 화면으로 자동 리다이렉트 (Next.js middleware)

### 4.2 관리자 로그인 (`/admin/login`)

- Supabase Auth 이메일/비밀번호 로그인
- 계정은 본사가 Supabase 콘솔에서 발급 (자체 가입 없음)
- 로그인 성공 시 원래 접근하려던 관리자 화면으로 복귀
- 사이드바 로그아웃 버튼으로 세션 종료

### 4.3 대시보드 (`/admin/dashboard`)

- 타석 이용 현황 (Supabase `bays` 실시간 구독 연동): 이용 중 / 입장
  대기 / 사용 가능 / 점검 타석 수
- 확인 필요한 운영 알림, 빠른 실행(입장 세션 시작, 매장 준비 ON,
  타석 배정, 제어 로그 출력)
- 실시간 시계 (클라이언트 갱신)

### 4.4 매장/타석/장비/회원 관리 (실 DB CRUD)

| 화면 | 테이블 | 주요 필드 |
| --- | --- | --- |
| 매장관리 | `stores` | 매장코드, 매장명, 주소, 전화번호, 타석수, 운영상태 |
| 타석관리 | `bays` | 타석코드, 표시명, 상태(available/in_use/waiting/maintenance), 메모 |
| 장비관리 | `devices` | 장비코드, 이름, 종류, 상태, 최근 점검일 |
| 회원관리 | `members` | 닉네임, 전화 뒤4자리, 가입경로, 연령대, 메모 |

- 등록/수정/삭제가 Supabase에 즉시 반영
- 매장 타석수 변경 시 타석관리 레코드와 자동 동기화

### 4.5 예약관리 (`/admin/reservations`)

- 예약 목록 조회, 등록, 수정 (타석 코드로 배정, 상태 변경)
- 예약 수정 시에도 DB 배타 제약조건이 적용되어 겹침 배정 시
  한국어 오류 안내

### 4.6 무인제어 (`/admin/automation`)

- 자동화 시나리오 4종 카드: 예약 입장 준비 / 타석 이용 시작 /
  이용 종료 정리 / 마감 전체 OFF
- 시흥점 쇼룸 실기기 테스트 흐름 (헤이홈·Tapo 장비)
- Home Assistant 스크립트 직접 호출 테스트 패널 (테스트 비밀번호 보호)
- 출입문 제어는 안전 검토 후 2차 기능으로 제외

### 4.7 기타 화면

경기기록, 랭킹, 대회운영, 조인모집, 리포트 화면은 1차 MVP에서
화면 구조와 데이터 모델(테이블)까지 구현된 상태이며, 실 데이터
연동은 2차 확장 범위이다.

## 5. 서버 API

| 경로 | 방식 | 기능 | 보호 |
| --- | --- | --- | --- |
| `/api/automation/reservation-prepare` | POST | 예약 5분 전 매장 장비 자동 준비 (cron 호출용) | Bearer 시크릿 (`CRON_SECRET`/`IOT_WEBHOOK_SECRET`) |
| `/api/automation/bay` | POST | 타석 단위 장비 제어 | 시크릿 검증 |
| `/api/automation/test` | POST | 쇼룸 장비 테스트 제어 | 테스트 비밀번호 |
| `/api/device/control` | POST | 장비 제어 웹훅 | 시크릿 검증 |

자동 준비 처리 결과는 `reservations.automation_prepare_status`
(pending/success/failed/skipped)와 `automation_prepared_at`에 기록한다.

## 6. 비기능 요구사항

- 한국어 UI 전용, Asia/Seoul 타임존 고정
- 개인정보 최소 수집: 이름/닉네임 + 전화 뒤 4자리만
- 관리자 화면 전체 인증 필수
- 데이터 접근 제어: Supabase RLS + 컬럼 단위 권한
  (익명 키로는 예약자 개인정보 컬럼 조회 불가)
- 반응형: 관리자 화면 PC 우선, 고객 화면 모바일 우선(최대폭 카드)
